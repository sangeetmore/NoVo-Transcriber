# NoVo Transcriber

<p align="center">
  <strong>novo transcriber</strong><br />
  Watch any video. Get clean notes.
</p>

NoVo Transcriber is a desktop learning agent for educational videos. It runs as a small floating Electron overlay while you watch a video, captures local audio and screen frames, indexes the live stream using Deepgram (STT) and multimodal LLMs (Groq, Ollama, or OpenAI), curates structured study notes, attaches screenshots, and writes the result into a live Notion document.

## What It Does

NoVo Transcriber turns a video-watching session into a structured Notion study guide in near real-time:

- Starts a native audio capture session from a lightweight Electron overlay.
- Streams WebM/Opus audio natively from your system to the local Go backend.
- Transcribes audio instantly via Deepgram WebSockets.
- Periodically captures desktop screenshots using native OS APIs to analyze visual context.
- Orchestrates a Curator agent that turns rolling event windows into structured Notion notes.
- Uploads visual evidence (screenshots) to Notion alongside the curated notes.
- Shows a live Agent Activity feed and mic input metrics right in the overlay.

## Architecture

The Electron overlay drives a lightweight Go backend. The React frontend directly captures your microphone/system audio using the browser `getUserMedia` API and streams WebM binary chunks over a WebSocket to the Go server. 

The Go backend funnels the audio stream directly into Deepgram for low-latency Speech-to-Text, periodically captures your screen, and feeds the combined multimodal context (transcripts + frames) into an LLM Curator Agent (powered by Groq, Ollama, or OpenAI) which evaluates the context and writes structured notes to Notion.

## Tech Stack

- **Desktop Overlay**: Electron + React + Vite.
- **Backend Server**: High-performance Go (Golang) WebSockets.
- **Audio Capture**: Browser-native `MediaRecorder` API (WebM/Opus).
- **Transcription**: Deepgram Streaming WebSocket API.
- **Intelligence**: Unified LLM Adapter supporting Groq (fast cloud), Ollama (fully local, privacy-first), and OpenAI.
- **Output**: Notion API for page creation, block writing, and image uploads.

## Setup

Clone the repository and move into the project root:

```bash
git clone <your-fork-or-repo-url>
cd NoVo-Transcriber
```

### 1. Environment Variables

Create a `.env` file in the project root:

```bash
cp .env.example .env
```

Fill in your configuration:

```env
# Notion Setup (Required)
NOTION_TOKEN=secret_your_notion_integration_token
NOTION_PARENT_PAGE_ID=your_notion_page_uuid_or_url

# Transcription (Required)
DEEPGRAM_API_KEY=your_deepgram_api_key

# LLM Provider (Choose ONE)
# For Groq:
GROQ_API_KEY=your_groq_api_key
# For Ollama (Local):
OLLAMA_BASE_URL=http://localhost:11434/v1
# For OpenAI:
OPENAI_API_KEY=your_openai_api_key
```

### 2. Backend Setup (Go)

Make sure you have [Go](https://go.dev/doc/install) installed.

```bash
cd backend-go
go mod tidy
go build ./...
```

### 3. Frontend Setup (Electron)

Make sure you have Node.js and NPM installed.

```bash
cd app
npm install
```

## Running The App

You need to run both the Go backend and the Electron frontend simultaneously.

### 1. Start the Go Backend

```bash
cd backend-go
go run cmd/server/main.go
```

You should see logs indicating the server started on `0.0.0.0:8000` and which LLM provider it auto-detected.

### 2. Start the Electron App

In a separate terminal:

```bash
cd app
npm run dev
```

### 3. Run a session

1. Ensure your speakers are turned up (if you're using a physical microphone) or you have correctly configured PulseAudio/PipeWire to monitor your system audio.
2. Open an educational YouTube video or local lecture.
3. Click **Start Session** in the NoVo Transcriber overlay.
4. Allow microphone permissions if prompted.
5. Watch the **Mic Input** meter in the UI. If it's bouncing, audio is successfully streaming!
6. You will see real-time transcripts appear in the Agent Activity feed.
7. Open the Notion link from the overlay to watch the Curator write notes in real-time.
8. Click **Stop Session** when done to finalize the document.

## Note for Linux Users

On Linux, browsers capture the default physical microphone. To capture the system audio (the video you are playing) without relying on your microphone picking up your speaker output:

1. Start the session in the app.
2. Open `pavucontrol` (PulseAudio Volume Control).
3. Go to the **Recording** tab.
4. Find the "NoVo Transcriber" application stream and change the capture source from your microphone to **"Monitor of Built-in Audio"** (or whatever your primary output is called).
5. The Mic Input meter in the app will instantly jump, and system audio will flow directly to Deepgram!

## Current Limitations

- Notes are generated in rolling windows (~15-30s), so adjacent sections can sometimes be slightly repetitive depending on the LLM used.
- Local screenshot capture uses OS-native calls which may require Screen Recording permissions on macOS.

## Future Scope
